# はぜくん3号 Step1 + 作戦ボード - しゃべる交代タイマー

5人フットサルチーム向けの、Web版しゃべる交代タイマーです。

## 機能

- 試合時間の設定
- 交代間隔の設定
- メンバー・ゴレイロ交代順の設定
- 残り時間の大きな表示
- 現在のゴレイロ、次のゴレイロの表示
- 交代タイミングのメンバー名読み上げ
- 交代タイミングの音声再生
- 残り1分の音声再生
- 試合終了の音声再生
- 手動交代
- 一時停止、再開、リセット
- 試合中ログの画面内表示
- 画面スリープ対策 Wake Lock API 対応ブラウザのみ

## 音声ファイル

以下のファイルを `public/audio/` に配置してください。

```text
public/audio/start.mp3
public/audio/change.mp3
public/audio/last-1min.mp3
public/audio/finish.mp3
```

## ローカル実行

```bash
npm install
npm run dev
```

## ビルド

```bash
npm run build
```

ビルド結果は `dist/` に出力されます。

## Azure Static Web Apps の設定目安

- App location: `/`
- Api location: 空欄
- Output location: `dist`

## 注意

このアプリはStep1向けのMVPです。
DB、localStorage、IndexedDBは使っていません。
設定やログは、画面を開いている間だけ保持されます。
ブラウザを更新するとリセットされます。


## デフォルトメンバー

メンバー・交代順には、初期値として以下を設定しています。

```text
くまちゃん
あっきー
ざきさん
うっち～
エド
```

## 交代時の読み上げ

メンバー欄に入力された全員について、交代タイミングでブラウザの音声合成により `次は、メンバー名` と読み上げた後、`public/audio/change.mp3` を再生します。

例:

```text
次は、くまちゃん
↓
change.mp3
```

ブラウザが音声合成に対応していない場合は、読み上げをスキップして `change.mp3` のみ再生します。


## ゲストを追加した場合

メンバー欄にゲスト名を追加した場合も読み上げ対象になります。

例:

```text
くまちゃん
あっきー
ざきさん
うっち～
エド
ゲスト太郎
```

この場合、ゲスト太郎の交代タイミングでも `次は、ゲスト太郎` と読み上げてから `change.mp3` を再生します。


## 見た目のトーン

本版は「はぜくん2号」として、1号よりも背景を少しだけ明るいダークトーンに調整しています。

将来的には、はぜくん10号に向かって段階的に背景を明るくしていく想定です。


## iPhoneで音が鳴らない場合

iPhoneのSafari/Chromeでは、タイマー処理からの初回音声再生がブロックされることがあります。

この版では、試合開始前に必ず `音声準備` ボタンを押す仕様にしています。

`音声準備` を押すと、現在の次のゴレイロ名を読み上げたあと、`change.mp3` を一度再生します。
これにより、iPhone側に音声再生を許可させてから試合開始できます。

推奨手順:

```text
1. メンバーと試合時間を設定
2. iPhoneの音量・マナーモードを確認
3. 音声準備を押す
4. 名前読み上げとchange.mp3が鳴ることを確認
5. 試合開始を押す
```


## iPhone/LINE内ブラウザで音声準備が失敗する場合

LINEアプリ内ブラウザでは、iPhoneのSafari/Chromeよりも音声再生が不安定になる場合があります。

本版では、`音声準備` を押した時点で `試合開始` は押せるようにしています。
`音声準備` で音が聞こえない場合は、画面右上の共有ボタンからSafariで開き直してください。

推奨手順:

```text
1. LINE内ではなくSafariでSWAのURLを開く
2. 音量・マナーモードを確認
3. 音声準備を押す
4. 試合開始を押す
5. 交代タイミングで鳴らない場合は、手動交代を一度押す
```


## はぜくん3号の追加機能: 作戦ボード

トップの画面切り替えから `作戦ボード` を開けます。

作戦ボードでは、以下を操作できます。

- フットサルコートを背景表示
- 自チームを黒丸で表示
- 相手チームを白丸で表示
- ボールを黄色の丸で表示
- 自チーム、相手、ボールをドラッグまたはタッチ操作で移動
- メンバー欄に入力した名前を黒丸に表示
- タイマー画面に戻っても、ブラウザを開いている間は配置を維持

## 保存方針

作戦ボードの配置はReactのstateだけで保持しています。
localStorage、IndexedDB、DB、APIは使っていません。

そのため、以下の場合は配置が初期化されます。

- ブラウザ更新
- タブを閉じる
- アプリを開き直す
- iPhone側でブラウザがメモリ解放される

## GitHub Actions / SWA 設定

Viteアプリなので、Azure Static Web Apps のGitHub Actionsでは以下を指定してください。

```yaml
app_location: "/"
api_location: ""
output_location: "dist"
```
